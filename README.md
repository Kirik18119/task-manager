# Task manager

Проект по управлению задачами

## Описание

В данном проекте должны быть реализованы возможности: регистрация (обычный пользователь), авторизация (обычный пользователь или администратор), создание пользователей(администратор), назначение категории пользователям(администратор), удаление пользователей(администратор), создание задач(администратор), редактирование задач(администратор), удаление задач(администратор), измение статуса задач(обычный пользователь), прикрепление файлов к задачам(администратор) с возможностью их последующего скачивания(обычный пользователь), добавление комментариев к задачам(обычный пользователь или администратор), добавление выполненных работ по задаче(обычный пользователь)  

## Детализация

### Зависимости

* PHP >= 8.2
* MySQL
* Composer
* Twig (для шаблонов фронтенда)

### Архитектурные требования

* Использование namespaces, composer autoload
* Создание кастомного Router, Request, Response, Model, Controller, View и остальных необходимых базовых классов
* Использование ООП принципов
* Создание дампа спроектированной базы данных
* Cоздание входного файла index.php, который будет обрабатывать все входящие на сервер запросы и вызывать соответствующи обработчики
* Создание консольной команды для генерация в базе записи администратора с заранее определенными значениями полей



## Инфологическое описание таблиц в базе данных (task_manager)

* users
    * id
    * first_name
    * last_name
    * is_admin
    * category (backend | fronted | testing)
* tasks
    * id
    * name
    * description
    * status (To-Do | In work | Code review | Testing | Done)
    * estimated_hours
    * created_at
    * deleted_at
* task_progresses
    * id
    * task_id (ссылка на таблицу tasks по id)
    * user_id (ссылка на таблицу users по id)
    * name
    * spent_hours
* task_files
    * id
    * task_id (ссылка на таблицу tasks по id)
    * file_path
* task_status_history
    * id
    * task_id (ссылка на таблицу tasks по id)
    * old_status (To-Do | In work | Code review | Testing | Done)
    * new_status (To-Do | In work | Code review | Testing | Done)
    * created_at
* task_assigment_history
    * id
    * task_id (ссылка на таблицу tasks по id)
    * old_user_id (ссылка на таблицу users по id)
    * new_user_id (ссылка на таблицу users по id)     
* comments
    * id
    * task_id (ссылка на таблицу tasks по id)
    * text
    * user_id (ссылка на таблицу users по id)
